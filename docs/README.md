# Allies Connect – Project Documentation

This folder contains supporting documentation for the **Allies Connect** platform.

---

## Contents

### Project Planning
- **Project Plan**  
  Outlines timeline, milestones, and overall development approach  

---

### Requirements & Analysis
- **Software Requirements Specification (SRS)**  
  Defines system requirements, user roles, and core features  

- **User Personas & Stories**  
  Describes target users and key usage scenarios  

- **User Flow Diagram**  
  Illustrates how users interact with the system  

---

### System Design
- **System Design Document**  
  Describes architecture, components, and design decisions  

- **Architecture Diagram**  
  Visual overview of system infrastructure  

- **Entity Relationship Diagram (ERD)**  
  Shows core database entities and relationships  

- **Data Dictionary**  
  Detailed definitions of database tables and fields  

---

### Implementation & Technical Documentation
- **API Documentation (Swagger)**  
  Describes available backend endpoints and usage  

---

### Testing
- **Test Documentation**  
  Includes testing strategy, unit tests, and integration test results  

---

## Notes

- The ERD focuses on core entities; additional supporting tables are documented in the Data Dictionary  
- API documentation is also available through the live Swagger interface
